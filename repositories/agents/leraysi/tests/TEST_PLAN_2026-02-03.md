## ⚠️ LECCIÓN APRENDIDA: Timezone Argentina → UTC

**PROBLEMA**: odoo-mcp enviaba `fecha_hora` sin convertir de hora local Argentina (UTC-3) a UTC. Odoo interpretaba "14:00" como UTC y mostraba "11:00" al usuario.

**SÍNTOMAS**:
- Vista de `salon.turno` mostraba hora incorrecta (3h menos)
- Calendario mostraba hora correcta (porque `confirmar-pago-completo` ya tenía fix)
- Bot WhatsApp mostraba hora correcta (usaba hora original)

**SOLUCIÓN**:
```typescript
// En crear-turno.tool.ts y reprogramar-turno.tool.ts
private argentinaToUTC(fechaHora: string): string {
  const [datePart, timePart] = fechaHora.split(" ");
  const [year, month, day] = datePart.split("-").map(Number);
  const [hour, minute, second] = (timePart || "00:00:00").split(":").map(Number);

  // Sumar 3 horas para convertir Argentina (UTC-3) a UTC
  const date = new Date(Date.UTC(year, month - 1, day, hour + 3, minute, second || 0));

  // Formatear a Odoo
  return `${y}-${m}-${d} ${h}:${min}:${s}`;
}
```

**ARCHIVOS MODIFICADOS**:
- `crear-turno.tool.ts`: Agregado `argentinaToUTC()`, usar antes de guardar
- `reprogramar-turno.tool.ts`: Agregado `argentinaToUTC()`, usar en create y write
- `confirmar-pago-completo.tool.ts`: Removido `argentinaToUTC()` (ya no necesario)

**REGLA**: Toda fecha/hora que se envíe a Odoo debe estar en UTC.

## ⚠️ LECCIÓN APRENDIDA: Actualización de nodos n8n

**PROBLEMA**: Al actualizar un nodo de AI Agent en n8n con `n8n_update_partial_workflow`, las actualizaciones parciales pueden PERDER parámetros existentes.

**SOLUCIÓN**: Siempre incluir TODOS los parámetros requeridos en una sola actualización:

```json
{
  "promptType": "define",
  "text": "={{ $json.userPrompt }}",
  "options": {
    "systemMessage": "..."
  }
}
```

**NUNCA** actualizar solo `options.systemMessage` sin incluir `promptType` y `text`.

**Nodos afectados**:
- Master AI Agent-Main (ID: `a422fb8f-1188-4f40-a4ee-8e7679694d8d`)
- Calendar Agent (verificar ID si se necesita actualizar)

## ⚠️ LECCIÓN APRENDIDA: Mapeo de servicios a odoo_key

**PROBLEMA**: El nodo BuildAgentPrompt construía el JSON para las tools de odoo-mcp usando `data.servicio` (display name como "Corte mujer") en vez de `data.servicio_odoo_keys[0]` (odoo_key como "corte_mujer").

**ERROR**:
```
"servicio": "${data.servicio || 'otro'}"  // ❌ INCORRECTO - envía "Corte mujer"
```

**SOLUCIÓN**:
```
"servicio": "${data.servicio_odoo_keys?.[0] || 'corte_mujer'}"  // ✅ CORRECTO - envía "corte_mujer"
```

**FLUJO DE DATOS**:
```
ParseInput → calcula servicio_odoo_keys[] desde SERVICIOS_CONFIG
           → BuildAgentPrompt DEBE usar servicio_odoo_keys[0] para tools
           → odoo-mcp recibe el odoo_key correcto
```

**REGLA**: Cualquier campo que se envíe a odoo-mcp DEBE usar el formato odoo_key, NO el display name.
